## Основная информация
Приложение разработано на Android Studio Meerkat | 2024.3.1 Patch 1, built on March 13, 2025.
- Язык: Kotlin;
- MinSdk: 30;
- Архитектура: Clean architecture;
- Сеть: Retrofit + OkHttp как стандарт работы с сетью в Android;
- ORM: Room была выбрана из-за удобной интеграции с Flow;
- Di: Hilt был выбран из-за удобной интеграции с компонентами Android и проверки циклических зависимостей на этапе компиляции;
- Асинхронность: Coroutines + Flow были выбраны как нативный для Kotlin и самый удобный подход к выполнению асинхронных операций;
- Ui: XML + ViewBinding;
- Image download/cache: Picasso была выбрана как простая и достаточная для данного проекта библиотека;
- Навигация: NavComponent + SafeArgs были использованы для упрощения навигации и передачи данных между фрагментами приложения.

В приложении имеются два Gradle модуля App и Data. Domain слой не вынесен в отдельный модуль для упрощения из-за простой бизнес логики. В Data модуле имеются два источника данных Local и Remote, где Local является SSOT. Репозиторий же предоставляет данные для Ui в виде Flow. Хоть получение списка пользователей и происходит с помощью Flow, для отображения подробной информации об конкретном пользователе используется обычная suspend функция, поскольку Api не имеет возможности предоставлять изменения конкретного пользователя и такого подхода было достаточно. Исключительно для ускорения разработки сохранение в базу данных происходит в виде сериализации Network модели и сохранении ее как json в поле таблицы. Хоть это и лишает возможности делать запросы к отдельным полям, в данном задании этого не требуется. 

В Ui слое используется стандартный подход с XML + ViewBinding с Fragments и ViewModel, которые предоставляют полученные от репозитория данные для Ui в виде State/Shared Flow с классами данных uiState. В Ui слое дополнительно была реализована анимация между фрагментами, темная и светлая тема, локализация, хоть и без настроек внутри приложения. Список пользователей реализован с помощью RecyclerView с DiffUtil, который можно обновить с помощью SwipeToRefresh. В приложении реализованы неявные Intent для работы с email, адресом и номером телефона. Добавлена обработка ошибок сети и отсутствия данных.
